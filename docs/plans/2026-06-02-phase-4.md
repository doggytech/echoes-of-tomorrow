# Echoes of Tomorrow Phase 4 Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Make runs feel more personal and tactical by introducing meaningful inventory items, named NPC relationship tracks, and wound/recovery loops that directly alter future choices.

**Architecture:** Keep Phase 4 lightweight and deterministic with pure helper functions in `game.js` for item rewards, relationship deltas, wound changes, and choice augmentation. Feed the resulting inventory/relationship/wound state into `server.js` prompt context so AI scenes honor the same state, then surface the new state in the existing vanilla JS UI.

**Tech Stack:** Node.js, Express, vanilla JavaScript, node:test.

---

## Selected Phase 4 features
1. **Inventory items that matter** — discovered items now persist and can unlock special choices in later scenes.
2. **Named NPC relationship tracks** — recurring characters gain/lose affinity based on player actions and can unlock social leverage.
3. **Wound and recovery loops** — risky turns can inflict persistent wounds, while careful/recovery options can clear them and stabilize health.

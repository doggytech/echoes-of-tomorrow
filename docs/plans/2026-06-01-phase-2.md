# Echoes of Tomorrow Phase 2 Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Make Echoes of Tomorrow feel more reactive and persistent by giving choices mechanical consequences, preserving recurring world entities, and surfacing named story arcs.

**Architecture:** Keep the app as a lightweight vanilla JS + Express experience. Add pure frontend helpers for stat consequences, entity discovery, and arc labeling so the new behavior can be tested with `node:test` before wiring it into UI rendering and save summaries. Extend the AI prompt context with known factions and current arc so later scenes can reflect the player's evolving run.

**Tech Stack:** Node.js, Express, vanilla JavaScript, node:test.

---

## Selected Phase 2 features
1. **Stats that matter** — choices now change health, memory, and reputation in visible ways.
2. **Recurring NPC / faction memory** — discovered characters and factions persist across scenes and show up in the UI.
3. **Named story arcs** — the run exposes a current arc title and arc-specific objective so progress feels more authored.

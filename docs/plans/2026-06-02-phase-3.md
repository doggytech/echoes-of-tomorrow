# Echoes of Tomorrow Phase 3 Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Deepen the game's mid-run continuity by introducing recurring quest chains, stronger faction reactions, and branch-aware scene steering that keeps each run feeling distinct.

**Architecture:** Keep the game lightweight and deterministic by adding pure helper functions in `game.js` for faction standing updates, quest progression, and branch-scene guidance. Feed the resulting quest/faction context into `server.js` prompt construction and save summaries, then surface the state in the existing vanilla JS UI without changing the app architecture.

**Tech Stack:** Node.js, Express, vanilla JavaScript, node:test.

---

## Selected Phase 3 features
1. **Recurring quest / goal chains** — each major branch now has an advancing quest card with staged objectives.
2. **Stronger faction reactions** — discovered factions now accumulate standing based on the player’s actions.
3. **Branch-specific scene steering** — AI prompts and fallback scenes include branch-aware direction so runs diverge more clearly.
